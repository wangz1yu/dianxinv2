## 2. 文案锁定规则（25e2cf0）

### 2.1 禁止改动
- 任何文本内容 **不可改字**（包括标点、数字、全角/半角、按钮文字）
- 不新增承诺性文案（例如“更低”“更快”“行业领先”等）

### 2.2 允许的“结构性使用方式”
- 允许把原有句子拆成多行排版（不改字）
- 允许把已有文案“搬到另一个板块”作为新结构模块的内容（例如用案例中的 challenge 组成“痛点条带”）

### 2.3（实现约束）建议的技术兜底
在实现阶段把首页文案抽离到 `src/content/home.copy.ts`（只导出常量），各 section 只引用常量，避免后续 UI 调整时误改字符串。

## 4. 动效总策略（更“行业官网节奏”）

> 原则：减少“每块都弹”的碎动效，改成 **段落感** 与 **转场感**。

1) Section 入场：标题与分隔线先出现，内容后出现（统一曲线与时长）  
2) Problem Strip：横向条带按顺序 reveal（0.06~0.1s stagger）  
3) Demos：采用 sticky 分段滚动；在滚动到每段时给 demo 容器一个轻微的“翻页/切页”过渡（opacity + y + blur 轻量）  
4) Clients 条带：可以继续无限滚动，但速度更慢、遮罩更克制

4) 动效节奏整体更“行业官网”，不花哨但有质感
