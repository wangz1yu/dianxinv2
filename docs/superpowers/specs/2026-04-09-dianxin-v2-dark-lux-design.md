# 点薪云官网 v2 统一视觉体系（C：深色奢简 + 金色主题）Design Spec

> 目标：在 **不改变业务范围** 的前提下，让所有内页与各个 section 统一到同一套“深色奢简 + 权威克制”的视觉体系，避免通用 SaaS 卡片风、避免“AI 模板感”。
>
> 约束：使用公司主题色作为唯一高光：`#fbc16a`、`#fbc779`（可做渐变/层次，但不引入第二个强 accent）。

---

## 0. 不可变更项（Hard Requirements）

1) **首页所有交互与动效保留**：包括但不限于 Navbar 交互（滚动状态/下拉/移动端菜单）、各 section 的滚动进入动效、Services/Pricing/Qualifications/FAQ/Demo 等现有功能交互。  
2) **首页 Hero 保留“视频 + 原文案”**（方案 A）：继续使用视频作为首屏背景（`https://oss.dianxin.love/hero.mp4`），并保留以下文案结构与 CTA（仅做视觉体系统一与排版优化）：
   - 徽标文案：`智能用工新时代`
   - 标题：`灵活用工` / `智领未来`
   - 描述：`为企业提供一站式灵活用工解决方案，从用工匹配、智能结算到保险保障，助力企业降本增效，让用工更简单。`
   - CTA：下载解决方案、观看演示（链接保持不变）

---

## 1. 视觉主张（Visual Thesis）

**深墨背景 + 细金高光 + 版式秩序**。像一本“高端合规报告”而不是“卡片式营销站”。

关键词：
- 权威 / 安全 / 合规 / 风控
- 低噪声信息表达：更少色彩、更少组件外框、更清晰层级
- 质感来自：排版、留白、分隔线、图像处理、微动效

---

## 2. 设计 Tokens（颜色 / 字体 / 间距）

### 2.1 颜色（Dark Lux Palette）

**底色体系（背景/面板/边界）**
- `ink-950`：`#0b0d12`（页面主背景）
- `ink-900`：`#0f1420`（面板底）
- `ink-850`：`#121a28`（面板 hover / 叠层）
- `stroke`：`rgba(255,255,255,0.10)`（hairline 分隔/描边）
- `stroke-strong`：`rgba(255,255,255,0.18)`（少量需要强调的边）

**文字体系**
- `text`：`rgba(255,255,255,0.92)`（主文本）
- `text-muted`：`rgba(255,255,255,0.70)`（辅助说明）
- `text-dim`：`rgba(255,255,255,0.55)`（标签/元信息）

**品牌金高光（唯一 Accent）**
- `brand-gold-1`：`#fbc16a`
- `brand-gold-2`：`#fbc779`
- `brand-gold-grad`：`linear-gradient(90deg, #fbc16a 0%, #fbc779 55%, #fbc16a 100%)`

**使用规则（很重要）**
1) 金色仅用于：主 CTA、关键数值、关键徽标/标签、分隔线“擦亮”动效端点  
2) 除金色外，其余颜色尽量来自灰阶与透明度，不出现第二个强饱和色  
3) 图片统一“低饱和/偏暗”，必要时加轻微金色/暖色叠色，让全站图像一致

### 2.2 字体（Typography）

两套字体，分别承担“权威标题”与“理性正文”：
- 标题（Display / Serif）：`Noto Serif SC`（400/600/700）
- 正文（Sans）：`IBM Plex Sans`（300/400/500/600）

排版规则：
- H1/H2 使用衬线，H3 以下可用无衬线
- label 使用无衬线，字距加大（tracking 0.18–0.26em）
- 数字（KPI）可用无衬线 SemiBold，搭配金色或下方 hairline

### 2.3 网格与间距（Layout）

- Desktop 内容最大宽度：`max-w: 1120px`（允许 hero/图片 full-bleed）
- 基础间距单位：8px
- section 垂直节奏：`96px`（大段），`72px`（中段），`40px`（小段）
- 分隔线（hairline）：1px，透明度 0.10–0.18，尽量“少而准”

---

## 3. 信息架构映射（不改业务，但统一表达方式）

### 3.1 全站统一“页面骨架”

每个内页都遵循同一骨架：
1) **Page Header**（面包屑 / 大标题 / 一句话副标题 / 右侧 CTA 或锚点目录）
2) **Body Sections**（每段只有一个 job：解释/证明/深化/转化）
3) **Footer**（统一）

### 3.2 Section 的“统一模板”

每个 section 由这些元素构成（顺序固定，形成识别度）：
- `label`（小字母间距，例如 “COMPLIANCE / 风控体系”）
- `title`（衬线标题）
- `hairline divider`（标题下 1px 线）
- `content`（列表/图文/数据）
-（可选）`secondary action`（文字链接 + 细下划线动效）

禁止事项：
- 默认卡片网格（card mosaic）
- 每个信息都一个圆角卡片
- 多重渐变背景叠加（会变成“AI 风”）

---

## 4. 关键组件规范（跨页面复用）

### 4.1 Navbar

- 透明玻璃感：`bg: rgba(255,255,255,0.06)` + `backdrop-blur`
- 滚动后：更实、边界更清晰（stroke 增强），但仍保持克制
- hover：下划线拉伸（不是背景色块）

### 4.2 Button

- Primary：金色渐变（仅 1 个主按钮/屏）
- Secondary：outline（stroke-strong）+ hover 轻微提亮
- Link：文字 + 下划线从左到右展开

### 4.3 Cards / Panels（仅在“承载信息”时用）

面板不是为了“好看”，而是为了可读性：
- 面板：`ink-900`，边 1px stroke
- 圆角：12–16（不要过大）
- 面板内分隔：hairline

---

## 5. 动效体系（全站 3 个动作，不做花活）

1) **Hero Entrance**：标题/副标题/CTA 分段入场（y + opacity），节奏 600–900ms  
2) **Section Reveal**：标题与 divider “擦亮式”出现（opacity + scaleX）  
3) **Hover Highlight**：按钮高光掠过/链接下划线拉伸（200–300ms）

动效原则：
- 能被录屏一眼看出“质感”，但不喧宾夺主
- 所有动效曲线统一（建议 ease: [0.22,1,0.36,1]）

补充说明（与 Hard Requirements 对齐）：
- “全站 3 个动作”指的是 **新增的统一动效语言**；并不删除现有交互动效。现有动效保留，但会把其视觉呈现（颜色/阴影/分隔线/高光）统一到本设计体系。

---

## 6. 落地计划（经你确认后才执行）

推荐执行顺序（统一效率最高）：
1) 先实现 v2 设计系统（tokens + 基础组件：Navbar/Footer/Button/SectionHeader/PageHeader）
2) 首页所有 section 统一迁移到新模板
3) 内页按类别批量迁移：列表型 → 详情型 → 内容型
